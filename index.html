web各种攻击原理及防护

发表于 2020-09-24

web各种攻击原理及防护

SQL注入

原理：

sql注入原理是将sql语句代码伪装到输入参数中，传递到服务器进行解析并执行的一种攻击手法，把sql命令插入到web表单提交，最终达到欺骗服务器执行恶意的sql命令

在对服务端发起请求参数中插入sql代码，服务端在执行sql操作时，会对注入攻击sql语句进行拼接

sql注入攻击是指通过构造特殊的语句参数传入到web中，通过sql语句进而执行攻击者想要的操作，主要原因是没有过滤用户输入参数，导致非法数据入侵系统

防护：

  1. 对用户输入进行校验，对单引号或#号一些符号进行转换
  2. 不要使用管理员权限的数据库连接，为没有应用使用单独的权限有限的数据库连接
  3. 不要把机密信息直接存放，加密进行处理
  4. 尽量不要给出报错错误信息

文件上传

原理：

将构造好的恶意代码上传到web可访问的目录中，由于上传功能没有严格的限制和校验，从而使得恶意代码被服务器进行解析执行

绕过：

前端：利用前端JavaScript进行防御文件上传，可以进行修改JavaScript代码进行绕过，还可以进行burpsuite抓包，修改文件后缀名进行绕过

后端：MIME类型的检查，修改Conten-Type值，gif的MIME值为image/gif,上传1.php，然后修改Conten-Type值

黑名单绕过：黑名单定义了一系列不安全的扩展名，可以变换大小写如PHP或pHP绕过

利用黑名单中没有的扩展名进行绕过，如asa，cer等等

配合iis解析漏洞，iis6.0当建立asa，asp文件夹时，该文件夹目录下任意文件都将被当做asp文件来解析

白名单绕过：使用%00截断，%00相当于结束符

在图片内容插入webshell

防护：

  1. 检查文件上传路径（避免00截断，iis6.0文件夹解析漏洞，目录遍历）
  2. 检查文件后缀名
  3. 文件MIME验证，比如gif图片MIE为image/gif
  4. 文件内容检查，（避免图片插入webshell）
  5. 对图片进行二次渲染
  6. 文件进行重命名，比如随机字符，时间戳避免攻击者获得webshell路径

XSS跨站脚本攻击

原理:

xss跨站脚本攻击分为反射型xss和存储型xss，反射型是将代码注入到网页上在浏览器上触发，存储型是把代码保存在数据库或文件中，但web进行读取利用代码时即可触发，xss跨站脚本攻击盗取cookie信息，修改页面等等，

防护：

  1. 对特殊字符HTML进行实体转码
  2. 标签属性设置白名单

CSRF跨站请求伪造

原理：

CSRF攻击是利用网站对用户浏览器的信任，挟持用户当前已登录的web应用程序，去执行非本意的操作，

防护：

  1. 使用token进行验证，可以在http请求中以参数的形式加入一个随机token，服务器验证token，如果token不正确，则拒绝该请求
  2. 验证HTTP REFERER,该字段记录了http请求来源地址

SSRF服务器端请求伪造

原理：

SSRF是利用存在缺陷的web应用作为跳板来攻击远程和本地服务器，攻击者可以利用ssrf漏洞来获取内网系统信息，服务器端提供了从其他服务器获取数据的功能

危害：

  1. 对内外网进行端口和服务扫描
  2. 攻击内网本地程序
  3. 对内网web应用进行指纹识别，识别企业内部资产信息
  4. 利用file协议读取本地数据

防护：

  1. 过滤返回的信息，如果web是去获取某一类型文件，返回前先验证是否符合标准
  2. 限制请求的端口，如80,443，
  3. 禁止不常用的协议，如file,gopher,ftp等等
  4. 设置URL白名单或者限制内网IP

MySQL注入攻击

发表于 2020-08-19 更新于 2020-08-21

什么是注入攻击

服务器在接受客户端的查询参数后，未对查询参数进行严格过滤，导致恶意用户可以在查询参数中插入恶意sql语句来查询数据库敏感信息，最终导致数据库信息泄露。

攻击方式

  1. 联合查询注入
  2. 报错注入
  3. 布尔型盲注
  4. 延时盲注

判断是否存在注入漏洞

以DVWA靶场为例
输入单引号’


出现报错，可能存在sql注入

正常输入 1

SQL注入语句解析：
mysql> select first_name,last_name from dvwa.users;
mysql> select first_name,last_name from dvwa.users where user_id='1';
你输入1 相当于在最后的两个单引号中间插入一个1来执行mysql的查询语句

输入1’and’1’=’1 (1’和前面的单引号’闭合,’1和后面的单引号’闭合了)

SELECT first_name, last_name FROM users WHERE user_id = '1' and '1' ='1'

继续输入1’and’1’=’2
SELECT first_name, last_name FROM users WHERE user_id = '1' and '1' ='2'
没有返回任何画面

判断可能为字符型注入

联合查询

收集信息

获取当前数据库和数据库用户名

1' union select database(),user() #

显示得到当前的数据库名为dvwa,数据库用户名为root@localhost

获取数据库版本和操作系统版本

1' union select version(),@@version_compile_os#

为什么要获取数据

我们需要判断MySQL的版本。因为当MySQL的版本小于4.0时，是不支持union select联合查询的；
当MySQL版本大于5.0时，有个默认数据库information_schema，保存了 Mysql服务器所有数据库的信息,如数据库名，数据库的表，
表栏的数据类型与访问权限等。该数据库拥有一个名为 tables 的数据表，该表包含两个字段 table_name 和table_schema，分别记录 DBMS 中的存储的表名和表名所在的数据库。

获取数据表名

1' union select table_name,table_schema from information_schema.tables where table_schema='dvwa' #

在information_schema数据库库的表中查询有关dvwa数据库的表

这里获取到了有dvwa的数据库里有两张表guestbook和users

我们对users表进行查询获取表中的列名

1' union select 1,group_concat(column_name) from information_schema.columns where table_name='users' #

group_concat(column_name)函数可列举该字段的所有内容

获取users表中的数据user，password字段信息

# SQL注入危害
  1. 数据库信息泄露
  2. 网站挂马：当拿到webshell时，将木马挂在服务器上攻击别人
  3. 读取文件获取webshell

  # 如何防范
  1. 对特殊字符进行编码转换--比如开启gpc（默认开启）
  2. 不要使用管理员权限的数据连接，最好为每个应用使用单独的数据连接
  3. 尽量少给提示
  4. 使用安全产品

### 其他
联合查询：合并多个相似的选择查询结果

联合查询限制条件：必须由两条或两条以上的select语句组成，语句之间用关键字union分隔

                  union中的每个查询必须包含相同的列

不管是报错还联合注入都是基于整型和字符型注入这两种类型下的

报错注入：由于配置不当，错误信息被输出到前台，导致攻击者根据报错信息进行一系列操作

延时注入和布尔注入共同点：都是一个字符一个字符爆出来的，耗费时间长，都没有显示位，都要ASCII码推算
